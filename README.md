# trilha-tech-fin-blueprint
Pedagogical logic, technical workflow, and document DNA for CIEBP Tech-Fin project.
# 🛠️ Project Archive: Trilha Tech-Fin Workflow

**Owner:** Daniel Landi (Engineer & Math Professor)  
**Context:** CIEBP (Centro de Inovação da Educação Básica Paulista)  
**Goal:** Create a pedagogical journey for High Schoolers bridging Finance, Math, and Technology.

---

## 1. The "Persona & Tone" Protocol
To recreate the documents, the AI must balance three specific roles:
*   **The Engineer:** Focuses on hardware (Creality K1C, Arduino, CoreXY), code (Python/C), and industrial efficiency (BOM/ROI).
*   **The Mathematician:** Formalizes logic using LaTeX, focusing on compound interest, geometry, and scaling laws.
*   **The Pedagogue:** Uses "Escuta Ativa" (Active Listening) and socratic provocation to turn students from consumers into producers.

---

## 2. The Structural "Mão na Massa" Engine
Every activity document must follow this 4-step sequence:
1.  **Provocation/Theory (20 min):** A socio-technical hook (e.g., "Why does the bank deny credit based on your zip code?").
2.  **Practical Application (40 min):** Hands-on coding, design, or building (e.g., Python scripts for interest rates or 3D printing MVPs).
3.  **Innovation Challenge (20 min):** A "stress test" or business case (e.g., "The Pitch" or "The Break Test").
4.  **The Bridge (10 min):** Explicitly linking the current space to the next one in the innovation hub.

---

## 3. Key Technical Anchors
*   **Language:** Portuguese (Brazil) for pedagogical content; English/Technical for documentation.
*   **Math Formalization:** Standard LaTeX for formulas like $M = C(1 + i)^t$.
